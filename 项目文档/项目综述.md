# 第一部分：项目综述

## 一、项目背景与目标

本项目是一个功能完备的**全栈酒店管理系统**，采用前后端分离架构，旨在为酒店提供从用户预订到后台管理的一站式解决方案。系统分为**用户端（C端）**和**管理端（B端）**两大模块，覆盖酒店运营的核心业务流程。

### 核心设计目标

- 🎨 **优秀的用户体验**：简洁直观的界面，流畅的交互流程
- 📊 **智能数据分析**：空置率预测、实时统计、可视化决策支持
- 🏗️ **高性能架构**：数据预计算策略，提升查询效率
- 🔐 **安全的权限系统**：JWT认证、角色权限控制
- 🎯 **实时房态管理**：可视化时间轴展示入住情况

## 二、系统总体设计

### 2.1 系统架构

系统采用**前后端分离**架构，前端使用React构建单页应用（SPA），后端使用Spring Boot提供RESTful API服务，MySQL作为数据持久化层。

### 2.2 功能模块划分

#### 用户端模块（C端）
- **首页展示**：酒店介绍、品牌形象
- **房型浏览**：房型列表、搜索过滤、会员价格
- **房型详情**：VR全景预览、日期选择、在线预订
- **订单管理**：我的订单、支付、取消、退款

#### 管理端模块（B端）
- **房型管理**：房间数量调整、房间实例CRUD
- **订单管理**：订单确认、入住办理、退房结算、退款审批
- **数据分析**：空置率统计、趋势预测、可视化图表
- **入住规划**：时间轴展示、房态监控

## 一、技术选型与路线

### 1.1 前端技术栈

| 技术 | 版本 | 选型理由 |
|------|------|----------|
| React | 18.3.1 | 组件化开发、Hooks简化状态管理、生态完善 |

| ECharts | 5.5.0 | 强大的数据可视化、丰富的图表类型、性能优秀 |
| React Router | 6.26.2 | SPA路由管理、代码分割、嵌套路由支持 |
| axios | - | Promise-based HTTP客户端、拦截器支持 |

### 1.2 后端技术栈

| 技术 | 版本 | 选型理由 |
|------|------|----------|
| Spring Boot | 3.5.6 | 快速开发、自动配置、生态成熟 |
| MyBatis-Plus | 3.5.8 | 简化CRUD操作、代码生成器、性能优化 |
| MySQL | 8.0 | 关系型数据库、事务支持、成熟稳定 |
| JWT | 0.11.5 | 无状态认证、跨域友好、易于扩展 |
