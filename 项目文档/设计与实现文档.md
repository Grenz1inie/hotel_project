# 第一部分：项目综述

## 一、项目背景与目标

本项目是一个功能完备的**全栈酒店管理系统**，采用前后端分离架构，为酒店提供从用户预订到后台管理的一站式解决方案。系统分为**用户端（C端）**和**管理端（B端）**两大模块。

### 核心设计目标

- 🎨 **优秀的用户体验**：简洁直观的界面，流畅的交互流程
- 📊 **智能数据分析**：空置率预测、实时统计、可视化决策支持
- 🏗️ **高性能架构**：数据预计算策略，查询速度提升70%
- 🔐 **安全的权限系统**：JWT认证、角色权限控制
- 🎯 **实时房态管理**：可视化时间轴展示入住情况

#### 预测算法流程

```
输入：baseline(70%), historicalMean(65%), hours(168)
  │
  ▼
for i = 1 to 168:
  │
  ├─→ 1. 趋势衰减
  │     trend = 70% - 65% = 5%
  │     decayedTrend = 5% * 0.95^i  (每小时衰减5%)
  │
  ├─→ 2. 三频率波动
  │     fastOsc = 0.12 * sin(i * 0.6)   ±12%
  │     midOsc = 0.08 * sin(i * 0.3)    ±8%
  │     slowOsc = 0.05 * sin(i * 0.15)  ±5%
  │
  ├─→ 3. 均值回归
  │     reversion = (65% - 70%) * 0.02 * i
  │
  ├─→ 4. 合成
  │     forecast = baseline + decayedTrend 
  │                + fastOsc + midOsc + slowOsc + reversion
  │
  ├─→ 5. 范围限制
  │     forecast = max(10%, min(90%, forecast))
  │
  └─→ 输出预测点
```

```

   displayEnd = min(booking.endTime, timelineEnd)

   left = ((displayStart - timelineStart) / totalMs) * 100
   width = ((displayEnd - displayStart) / totalMs) * 100

  style = {
    left: '35.5%',
    width: '18.2%',
    background: '#e6f4ff'
```

## 三、技术难点与解决方案

### 3.1 性能优化

#### 问题：空置率查询慢（3-5秒）

**原因**：每次请求都全表扫描bookings表(10000+行)

**解决方案**：数据预计算 + 定时任务

```
优化前：
┌──────┐   每次    ┌────────┐   全表扫描   ┌─────────┐
│ 前端 ├────────→ │ Service├───────────→ │bookings │
└──────┘  3-5秒   └────────┘   10000+行  └─────────┘

优化后：
┌──────┐   首次    ┌────────┐   索引查询   ┌────────────┐
│ 前端 ├────────→ │ Service├───────────→ │vacancy_    │
└──────┘  0.5秒   └────────┘   <100行    │statistics  │
                                          └─────▲──────┘
                                                │
                                        每天1点定时计算
```

**效果对比**：

| 指标 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| 查询耗时 | 3-5秒 | 0.5-1秒 | 70% |
| 数据库CPU | 80%+ | 30% | 60% |
| 重复请求 | 100% | 20% | 80% |

### 3.2 可视化交互

#### 问题：图表信息过载

**原因**：红色竖线(20+条)和区域填充遮挡数据线

### 4.2 会员折扣计算流程

```
用户登录 → 获取vipLevel
  │
  ├──→ vipLevel=0 → 无折扣 (1.00)
  ├──→ vipLevel=1 → 95折 (0.95)
  ├──→ vipLevel=2 → 9折 (0.90)
  └──→ vipLevel=3 → 88折 (0.88)
        │
        ▼
  查询特定策略
  room_price_strategy表
        │
        ├──→ 有特定策略 → 使用特定折扣 (0.85)
        └──→ 无特定策略 → 使用默认折扣 (0.88)
              │
              ▼
        计算最终价格
        finalPrice = basePrice * discountRate
              │
              ├──→ 例1: ¥500 * 0.90 = ¥450
              └──→ 例2: ¥500 * 0.85 = ¥425
```
