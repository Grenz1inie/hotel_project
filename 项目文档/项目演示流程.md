# 🎯 酒店预订系统项目演示流程

> **演示目标**：全面展示系统的用户体验、业务功能、技术特性和管理能力  
> **演示时长**：15-20分钟  
> **演示环境**：前端 http://localhost:3000 + 后端 http://localhost:8080

---

## 📋 演示前准备

### 环境检查
- [ ] 后端服务已启动（Spring Boot on port 8080）
- [ ] 前端服务已启动（React on port 3000）
- [ ] 数据库连接正常（MySQL hotel_db）
- [ ] 浏览器已打开开发者工具（F12，用于展示网络请求）
- [ ] 准备两个测试账号：
  - 普通用户：`user` / `password`
  - 管理员：`admin` / `password`

### 演示数据
- 3家酒店（星河国际、海天假日、云栖温泉度假村）
- 9种房型（每家酒店3种）
- 测试用户账户（含会员等级和钱包余额）

## 🎬 第一阶段：系统介绍与首页展示（3分钟）

### 1.1 项目概述介绍
**说明内容**：
> "这是一个基于 Spring Boot + React + MySQL 的全栈酒店预订系统，采用前后端分离架构。前端使用 React 18 + Ant Design 5 构建现代化UI，后端使用 Spring Boot 3 + MyBatis-Plus 提供RESTful API。系统支持游客浏览、用户预订、会员体系、钱包支付、VR看房等完整业务流程。"

### 1.2 首页展示
**操作**：访问 `http://localhost:3000`

**展示重点**：
1. **视觉设计**
   - 英雄区域（Hero Banner）展示酒店主图和基本信息
   - 响应式布局和现代化UI设计
   - 星级评分展示（⭐⭐⭐⭐⭐）
   
2. **酒店信息**
   - 酒店名称：星河国际酒店
   - 位置：北京 · 朝阳区建国路99号
   - 介绍文案和特色标签

3. **交互设计**
   - 点击"立即探索房型"按钮
   - 平滑过渡动画

**讲解要点**：
> "首页采用沉浸式设计，Hero Banner 使用酒店实景图片作为背景，配合渐变蒙层和响应式布局。图片资源通过 Spring Boot 的静态资源服务提供，路径为 `/images/hotels/`。"

## 👤 第三阶段：用户注册登录与会员体系（3分钟）

### 3.1 用户注册
**操作**：点击右上角"登入"按钮 → 切换到"快速注册"标签

**展示重点**：
1. **注册表单**
   - 用户名（唯一性验证）
   - 密码（强度提示）
   - 确认密码（一致性验证）
   - 手机号（格式验证）
   - 邮箱（可选）

2. **表单验证**
   - 实时验证提示
   - 错误信息展示
   - 提交按钮状态控制

**讲解要点**：
> "注册功能支持完整的表单验证，包括用户名唯一性检查、密码强度校验、手机号格式验证等。后端使用 BCrypt 加密存储密码，确保用户信息安全。"

### 3.2 用户登录
**操作**：使用测试账号登录 `user` / `password`

**展示重点**：
1. **登录流程**
   - 输入账号密码
   - JWT Token 认证
   - 自动跳转回之前页面

2. **登录后状态变化**
   - 右上角显示用户名和会员等级
   - 显示"我的订单"和"个人中心"入口
   - 房型列表显示会员折扣价格

**讲解要点**：
> "系统采用 JWT (JSON Web Token) 进行身份认证。登录成功后，Token 存储在浏览器 localStorage 中，并在后续所有 API 请求的 Authorization Header 中携带。会员等级会影响房价折扣和退房时间等权益。"

### 3.3 会员体系展示
**操作**：观察房型列表的价格变化

**展示重点**：
1. **会员折扣**
   - 原价显示：¥299/晚
   - 会员价显示：¥269/晚（VIP 1级 9折）
   - 折扣标签和节省金额提示

2. **多级会员**
   - VIP 0级：基础用户（100% 原价）
   - VIP 1级：银卡会员（90% 折扣）
   - VIP 2级：金卡会员（85% 折扣）
   - VIP 3级：白金会员（80% 折扣）
   - VIP 4级：钻石会员（75% 折扣）

**讲解要点**：
> "系统实现了完整的会员等级体系，不同等级享受不同的折扣率。会员折扣策略由后端动态计算，支持全局基础折扣和房型专属折扣的组合。前端实时显示会员优惠信息，增强用户的价值感知。"

### 4.2 选择入住日期
**操作**：点击"选择入住时间"按钮

**展示重点**：
1. **日期选择器**

   - 双月份显示（桌面端）

   - 默认入住时间：今天 14:00

3. **时间调整**

4. **预抵时间（可选）**
   - 选择具体抵达时间（如 18:00）

**讲解要点**：
> "日期选择使用 react-date-range 库，提供直观的可视化日期范围选择。系统根据用户的 VIP 等级智能设置退房时间边界（VIP 0级：12点，VIP 4级：16点），这是会员特权的一部分。入住时间选择支持小时级别的精确控制。"

### 4.3 填写预订信息

2. **支付方式选择**

3. **钱包余额显示**
   - 当前余额：¥1000.00
   - 实时检查余额充足性
   - 余额不足时提示充值

4. **价格计算预览**
   - 房价：¥299 × 1晚
   - 会员折扣：-¥30（VIP1 9折）
   - 应付金额：¥269

**讲解要点**：
> "预订表单根据用户身份智能填充默认值，减少输入负担。钱包支付是推荐的支付方式，系统实时检查余额充足性。价格计算透明展示，包括原价、折扣和应付金额，增强用户信任感。"

### 4.4 提交预订
**操作**：点击"立即预订"按钮

**展示重点**：

2. **提交过程**
   - Loading 状态提示
   - 网络请求（在开发者工具中展示）
   - 成功/失败反馈

3. **成功响应**
   - 显示订单创建成功消息
   - 自动跳转到"我的订单"页面
   - 订单ID和详情展示

**讲解要点**：
> "预订提交时，后端会进行多重验证：库存充足性、时间段无冲突、用户余额充足（如果使用钱包支付）。成功后返回完整的订单信息，前端自动跳转到订单管理页面，形成闭环体验。"

## 👨‍💼 第六阶段：管理员功能演示（5分钟）

### 6.1 管理员登录
**操作**：退出当前账号 → 使用管理员账号登录 `admin` / `password`

**展示重点**：
1. **管理员标识**
   - 右上角显示"管理员"角色标签
   - 顶部导航栏增加"后台管理"入口

2. **权限区别**
   - 可以访问 `/admin` 管理后台

**讲解要点**：
> "系统实现了基于角色的访问控制（RBAC）。管理员账号拥有特殊权限，可以访问后台管理功能。路由守卫在前端拦截未授权访问，后端API也会验证JWT中的角色信息。"

### 6.2 库存管理
**操作**：进入 `/admin` 管理后台

**展示重点**：
1. **房型库存卡片**
   - 所有房型的库存统计
   - 当前房型名称和类型
   - 总房间数 / 可用房间数
   - 入住率进度条（视觉化展示）
   - 主题色标识（每个房型有独特颜色）

2. **库存调整操作**
   - 点击任意房型的"调整总数"按钮
   - 输入新的总房间数（如 10 → 15）
   - 提交调整
   - 实时更新卡片显示
   - 入住率自动重新计算

3. **入住率可视化**
   - 使用 Ant Design Progress 组件
   - 动态颜色：绿色（低）→ 黄色（中）→ 红色（高）
   - 百分比数字显示

**讲解要点**：
> "库存管理界面使用卡片式布局，每个房型的入住率通过环形进度条直观展示。调整库存时，系统会校验新的总数必须大于等于已被预订的数量，防止数据冲突。颜色渐变使用 HSL 色彩空间插值计算，随入住率动态变化。"

### 6.3 订单管理
**操作**：在管理后台向下滚动到"订单管理"部分

**展示重点**：
1. **订单筛选**
   - 酒店ID筛选
   - 房型ID筛选
   - 房间ID筛选
   - 用户ID筛选
   - 订单状态筛选（待确认/已确认/已入住/已退房）
   - 联系电话搜索
   - 日期范围筛选

2. **订单表格**
   - 完整的订单信息展示
   - 排序功能（按时间、金额等）
   - 分页控制（每页10条）

3. **订单操作**
   - **确认订单**：PENDING → CONFIRMED
   - **办理入住**：CONFIRMED → CHECKED_IN
   - **办理退房**：CHECKED_IN → CHECKED_OUT
   - **拒绝订单**：取消并退款
   - **删除订单**：硬删除（危险操作）

4. **批量操作演示**
   - 筛选"待确认"状态的订单
   - 点击"确认"按钮
   - 状态标签从"待确认"变为"已确认"

**讲解要点**：
> "订单管理提供了强大的筛选和批量操作功能。管理员可以根据多个维度筛选订单，然后执行确认、入住、退房等操作。订单状态流转：待确认 → 已确认 → 已入住 → 已退房。每个状态变更都会记录时间戳，并可能触发钱包余额变动（如退款）。"

**展示重点**：

## 🎓 第八阶段：技术亮点总结（2分钟）

### 技术架构
**前端**：
- ⚛️ React 18 + React Router 6（现代化SPA）
- 🎨 Ant Design 5（企业级UI组件库）
- 🔐 JWT认证 + Context API（状态管理）
- 📅 react-date-range（日期选择）
- 🌐 VR全景（Pannellum 360°看房）
- 📊 ECharts（数据可视化）

**后端**：
- ☕ Spring Boot 3 + Spring Security
- 🗄️ MyBatis-Plus（数据库ORM）
- 🔑 JWT（无状态认证）
- 🏗️ RESTful API设计
- 📦 静态资源服务（图片托管）

**数据库**：
- 🗃️ MySQL 8.0
- 📐 三级结构：酒店 → 房型 → 物理房间
- 🔄 支持库存管理和房态追踪

### 核心功能
1. ✅ **完整的预订流程**：浏览 → 选择 → 预订 → 支付 → 管理

3. ✅ **钱包支付系统**：充值 + 消费 + 交易记录
4. ✅ **VR看房功能**：360°全景沉浸式体验
5. ✅ **管理后台**：库存管理 + 订单管理 + 房态管理 + 数据分析
6. ✅ **响应式设计**：桌面 + 平板 + 移动全适配
7. ✅ **安全认证**：JWT + BCrypt + 角色权限控制
