# 智能酒店预订与管理系统

## 项目概览

**智能酒店预订与管理系统**是一款面向现代酒店行业的全栈Web应用，提供从房间浏览、在线预订、支付结算到后台管理的完整解决方案。系统采用前后端分离架构，注重用户体验与管理效率，融合了VR全景预览、智能会员体系、数据分析等创新功能。

---

## 🎯 系统定位

### 目标用户
- **C端用户**：个人旅客、商务出差人员、休闲度假客户
- **B端管理员**：酒店前台、客房管理员、运营经理、数据分析师

- ✈️ 商务差旅预订

## 🏗️ 技术架构

### 技术栈
```
前端：React 18 + Ant Design + React Router
后端：Spring Boot + MyBatis-Plus + MySQL 8.0
部署：Node.js + Maven + Tomcat
```

### 架构特点
- 🔄 **前后端分离**：独立开发、灵活部署
- 🔐 **JWT认证**：安全可靠的身份验证
- 📊 **RESTful API**：标准化接口设计
- 💾 **关系型数据库**：数据一致性保障
- 🎨 **组件化开发**：代码复用性强
- 📱 **响应式设计**：多端适配

### 系统模块
```
┌─────────────────────────────────────────────┐
│              前端应用层 (React)              │
├─────────────────────────────────────────────┤
│  首页  │ 房间列表 │ 房间详情 │ 个人中心     │
│  登录  │ 我的订单 │ 管理后台 │ VR预览       │
```

#### 2.4 空置曲线分析 ⭐
这是系统的**核心亮点功能**，提供数据驱动的库存管理和决策支持。

- 📊 **可视化图表**
  - 基于ECharts的折线图
  - 流畅的动画效果
  - 响应式设计

- 🔍 **多维度筛选**
  - **房型选择**：支持多选，对比不同房型
  - **时间粒度**：小时/天
  - **时间范围**：最近7天/最近30天/自定义
  - **数据指标**：空置数量/空置率/预订率
  - **阈值设置**：高位预警线/低位预警线

- 📈 **数据展示**
  - **实线**：历史实际数据
  - **虚线**：预测数据
  - **预警线**：红色（高位）、橙色（低位）
  - **预警区域**：阴影标注异常时段
  - **事件标记**：节假日、活动日期

- 🔮 **智能预测算法**（核心创新）
  ```
  预测值 = 基准值 + 线性趋势 + 周期性因子 + 随机波动
  
  - 基准值：最近N天的平均空置率
  - 线性趋势：简单线性回归计算增长/下降趋势
  - 周期性因子：
    * 周末（周五-周日）：× 0.85（空置率更低）
    * 工作日前期（周一-周二）：× 1.1（空置率更高）
  - 随机波动：正弦波模拟，振幅 = 标准差 × 30%
  ```
  
  **预测效果**：
  - ✅ 非水平直线，有趋势性
  - ✅ 体现周期性（周末/工作日）
  - ✅ 合理波动，接近真实场景
  - ✅ 预测未来7-14天

- ⚠️ **预警系统**
  - **高位预警**（空置率>70%）
    - 提示：需求不足
    - 建议：降价促销、推广活动
  - **低位预警**（空置率<20%）
    - 提示：供不应求
    - 建议：提价、增加房源、限制预订

- 📊 **预警列表**
  - 显示所有预警事件
  - 房型、日期、空置率、建议措施
  - 按严重程度排序

- 📄 **报表导出**

## 📊 数据库设计

### 核心数据表

#### users（用户表）
```sql
- id: 用户ID（主键）
- username: 用户名
- password: 密码（加密）
- phone: 手机号（唯一）
- email: 邮箱
- role: 角色（USER/ADMIN）
- vip_level: VIP等级（0-4）
- total_consumption: 累计消费金额（DECIMAL(12,2)）
- created_at: 创建时间
- updated_at: 更新时间
```

#### room_types（房型表）
```sql
- id: 房型ID（主键）
- name: 房型名称
- price: 基础价格
- area: 房间面积
- bed_type: 床型
- max_occupancy: 最大入住人数
- total_count: 总数量
- available_count: 可用数量
- description: 描述
- amenities: 设施（JSON）
- images: 图片URL（JSON）
```

#### rooms（房间实例表）
```sql
- id: 房间ID（主键）
- room_type_id: 房型ID（外键）
- room_number: 房间号（唯一）
- floor: 楼层
- status: 状态（0-5）
- created_at: 创建时间
- updated_at: 更新时间
```

#### bookings（订单表）
```sql
- id: 订单ID（主键）
- user_id: 用户ID（外键）
- room_type_id: 房型ID（外键）
- room_id: 房间ID（外键，可空）
- check_in_date: 入住日期
- check_out_date: 退房日期
- room_count: 房间数量
- guest_count: 入住人数
- total_price: 总价
- status: 状态（PENDING/CONFIRMED等）
- contact_name: 联系人
- contact_phone: 联系电话
- created_at: 创建时间
- updated_at: 更新时间
```

#### payments（支付表）
```sql
- id: 支付ID（主键）
- booking_id: 订单ID（外键）
- user_id: 用户ID（外键）
- amount: 金额
- payment_method: 支付方式（WALLET/ONLINE等）
- status: 状态（SUCCESS/PENDING等）
- transaction_id: 交易流水号
- created_at: 创建时间
```

#### wallets（钱包表）
```sql
- id: 钱包ID（主键）
- user_id: 用户ID（外键，唯一）
- balance: 余额（DECIMAL(12,2)）
- created_at: 创建时间
- updated_at: 更新时间
```

#### wallet_transactions（钱包交易表）
```sql
- id: 交易ID（主键）
- wallet_id: 钱包ID（外键）
- user_id: 用户ID（外键）
- amount: 金额
- type: 类型（CHARGE/PAYMENT/REFUND）
- balance_after: 交易后余额
- description: 描述
- reference_id: 关联订单号
- created_at: 创建时间
```

#### vr_scenes（VR场景表）
```sql
- id: 场景ID（主键）
- room_type_id: 房型ID（外键）
- scene_name: 场景名称
- scene_url: 场景URL
- preview_image: 预览图
- sort_order: 排序
- created_at: 创建时间
```

## 🎓 技术亮点总结

### 🌟 1. VR全景预览技术
**创新点**：
- 业内领先的360度全景房间展示
- 热点交互增强用户沉浸感
- 显著降低"图片与实物不符"导致的退订率

**技术实现**：
- 前端：React + 自定义VR组件
- 资源：全景图片 + JSON配置热点
- 交互：拖拽旋转 + 点击热点

### 🌟 2. 智能VIP会员体系
**创新点**：
- 完全自动化的升降级机制
- 退款回减消费，公平合理
- 多级折扣体系促进用户消费

**技术实现**：
- 支付时：`total_consumption += amount` → 检查阈值 → 自动升级
- 退款时：`total_consumption -= amount` → 检查阈值 → 自动降级
- 数据库触发器 + 后端业务逻辑双重保障

### 🌟 3. 空置曲线智能预测
**创新点**：
- 基于历史数据的趋势预测
- 考虑周期性（周末效应）
- 高低位预警系统辅助决策

**技术实现**：
```java
// 简化的预测算法
double prediction = baseValue 
                  + linearTrend * day
                  + periodicFactor
                  + Math.sin(day * 0.5) * volatility * 0.3;
```

### 🌟 4. 实时数据同步机制
**创新点**：
- 页面可见性API监听标签页切换
- 自动刷新用户信息（VIP等级、余额、累计消费）
- 多端数据一致性保障

```javascript
document.addEventListener('visibilitychange', () => {

  }
```
